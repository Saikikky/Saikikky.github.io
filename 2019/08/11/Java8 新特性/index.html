Lambda表达式

Lambda表达式，也称作闭包，允许把函数作为一个方法的参数，使代码变得更加简洁。

带有参数变量的表达式，可以传递。

语法

lambda表达式的语法格式如下

方法体：可以是表达式或者代码块，是函数式接口里方法的实现，如果负责运算的代码无法用表达式表示，也可以使用代码块实现，但必须用{}包围并明确使用return语句。

  • 可选类型声明：不需要声明参数类型，编译器可以统一识别参数值。
  • 可选的参数圆括号：一个参数无需定义圆括号，但多个参数需要定义圆括号。
  • 可选的大括号：如果主体只包含一个语句，就不需要使用大括号。
  • 可选的返回关键字：如果主体只有一个表达式返回值则编译器会自动返回值，大括号需要指定明表达式返回了一个数值。
  • 示例

Lambda表达式主要用来定义行内执行的方法类型接口，例如，一个简单方法接口。

免去使用匿名方法的麻烦，并且给予Java简单但是强大的函数化编程能力。

变量作用域

lambda表达式只能引用标记了final的外层局部变量，不能在lambda内部修改定义在域外的局部变量。

  • 何时使用

    在需要显示创建函数是接口对象的地方都可以使用，lambda表达式必须要和Functional Interface一起使用，主要是用于替换内部匿名类和各种回掉，比如事件响应器和传入Thread类的Runnable等。

  • 公共定义的函数式接口 java.util.function

    • 功能性接口：Function<T, R>

      有输入参数，有返回值

      是对接收一个T类型参数，返回R类型的结果的方法的抽象

      通过调用apply方法执行内容

      需求：给定一个字符串，返回字符串长度

  • 消费型接口：Consumer

    有输入参数，没有返回值，一般 Consumer 接口伴随一些期望状态的改变或者事件的发生

    forEach 就是使用的 Consumer 接口虽然没有任何的返回值，但是像控制台输出结果

  • 供给型接口：Supplier

    没有参数传入，有返回值

    使用 get()方法获得这个返回值

  • 断言型接口：Predicate

    有传入参数，也有返回值 Boolean

    多用于判断与过滤，使用 test()方法执行这段行为

优点
  • 极大的减少代码冗余，同时可读性也好过匿名内部类。
  • 与集合类批处理操作，实现内部迭代。
与匿名内部类的区别
  • 在lambda中，this不是指向lambda表达式产生的对象而是外部对象。

stream 流

使用 for 循环的时候代码冗余且复杂，实现并行计算也很麻烦。

stream 是一个更高级的 Iterator，不像 Iterator 只能显示一个个的遍历元素并对其执行操作

stream给出操作就可以隐式的在内部进行遍历而且可以并行化操作，提高编码效率和程序可读性

  • forEach

  • collect(toList())

    stream 中很多操作都是惰性求值，因此调用 Stream 一系列方法之后还需调用一个类似 collect 的及早求值方法返回集合。

    及早求值就是在约束到变量的时候求值，惰性求值并不立即返回结果

  • map

    如果需要将一种类型的值转换成另外一种类型

    map操作可以将一个流中的值转换成一个新的流

  • filter

    筛选元素 遍历数据并检查其中的元素，获取字符串 List 中以数字开始的字符集合

  • flatMap

    可用Stream 替换值，然后将多个 Stream 连接成一个 Stream，生成一个 stream 对象取而代之

  • max和 min

  • reduce

    通过指定的函数把 stream 中的多个元素汇聚成一个元素

    reduce(初始值,(r,i)->r+i)或者 reduce((r,i)->r+”*”i)
