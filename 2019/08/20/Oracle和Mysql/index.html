Oracle和MySQL

Posted on 2019-08-20

Oracle和MySQL

  • Oracle是大型数据库而MySQL是中小型数据库，Oracle不开源价格昂贵，Mysql开源。

  • SQL语法不同，MySQL比较灵活而Oracle比较严格，比如单引号使用，分页的处理以及主键的自动增长。

  • 关于事务，MySQL默认对事务不支持，但innodb存储引擎支持，Oracle对事务完全支持。

  • MySQL是单进程多线程，Oracle是多进程(Windows下也是单进程)，在Windows操作系统下只能使用一个用户，或者是在不同的用户之间做切换，但在Linux下，是多用户或者相同用户可以同时连接操作系统。

  • 数据库和实例以及用户之间的关系。

    MySQL的实例是用户登陆时系统分配给用户的，而用户必须是先在MySQL中创建好，然后登陆用户mysql -u username -p。instance>database

    Oracle的实例是在创建数据库的时候就默认创建好的，而用户基于数据库实例，实例之间可以没有关系所以其中的用户也不尽相同，登陆不同的实例相当于登陆了不同的数据库。

    instance=database

  • SQL语法

    • 建表时，Oracle不支持在建表的同时增加字段注释，需要单独加注释

      MySQL可以同时增加字段注释

    • 自动增长的数据类型处理

      MySQL有自动增长的数据类型，插入记录时不用操作此字段，会自动获得数据值。Oracle没有自动增长的数据类型，需要建立一个自动增长的序列号，插入记录时要把序列号的下一个赋值给该字段。

    • 单引号的处理

      MySQL中可以用双引号包含字符串，Oracle只能用单引号包起字符串。在插入和修改字符串之前必须要做单引号的替换：把所有出现的一个单引号替换成两个单引号。

    • 分页

      MySQL中用LIMIT开始位置，记录个数，Oracle中每个结果集只有一个ROWNUM字段标明它的位置。

  • 索引

    索引是关系数据库中用于存放每一条记录的一种对象，主要目的是加快数据的读取速度和完整性检查。建立索引是一项技术性要求高的工作，一般在数据库设计阶段与数据库结构一道考虑，应用系统的性能与索引的合理直接有关。

    Oracle和MySQL的常用索引结构基本都是B树。
