for循环中创建对象和for循环外创建对象

  • 示例：

和

Object obj 是创建对象引用，引用的实例地址。 new Object() 是创建对象实例
两者都会占用系统资源。
改进之前for循环中创建了10000次引用和10000实例，改进后是创建了1次引用，10000实例。
在方法执行完后内存资源会被回收

这两种创建对象的方法究竟有区别吗？？

  • 每次循环声明创建的变量，在本次循环结束之后即会被标记为“可以被GC回收了”。即使Java的垃圾回收线程优先级相当低，也许在被标记为“可以被回收”后，这些内存空间并不能马上被真正的释放。

  • 声明在循环体外部的变量，人为地将其的声明周期拉长了。并不是什么好事。回收的时间被推后了不说，更将严重的后果可能是，原本就应该“不可达”的引用被有意无意的再次使用了。这是非常糟糕的事，也许数据就这么被破坏了，这种隐性的bug将大大增加。

  • Java的“foreach”循环。这个循环的写法例如for(Person p : personList)，明显的这就是使用的在循环体内声明变量的方式。
  • 对于基本数据类型，如int，声明在循环内和循环外，占用的都是栈内存，对真个内存管理控制意义不大。对于引用数据类型，写在循环内部确实多次开辟了内存，new出对象，消耗了内存。
